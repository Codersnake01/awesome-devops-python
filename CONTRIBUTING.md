# 📝 Guía de Contribución

Gracias por tu interés en contribuir a **Awesome DevOps + Python**. Este documento te guiará sobre cómo hacerlo.

## 🤝 Tipos de Contribuciones

### 1. Reportar Issues
Si encuentras un error, repositorio desactualizado o información incorrecta:
- Abre un **Issue** describiendo el problema
- Incluye toda la información relevante (URL, descripción, contexto)
- Sé específico y claro

### 2. Sugerir Nuevos Repositorios
¿Encontraste un excelente repositorio DevOps + Python?
- Asegúrate de que cumple con los criterios:
  - ✅ Está escrito en **Python** (o principalmente)
  - ✅ Relacionado con **DevOps**
  - ✅ Es **open-source** y activo
  - ✅ Tiene buena documentación
  - ✅ Tiene comunidad/stars significativos

- Abre un Issue con:
  - Nombre del repositorio y enlace
  - Por qué debería incluirse
  - Categoría sugerida
  - Número de stars (opcional)

### 3. Mejorar Descripciones
¿Puedes hacer la descripción más clara o exacta?
- Sugiere cambios en un Issue
- O haz directamente un Pull Request con mejoras

### 4. Agregar Secciones Nuevas
¿Hay una categoría importante que falta?
- Propón nuevas categorías (ej: "Logging", "Testing", etc.)
- Sugiere repositorios que se ajusten

## 🚀 Cómo Hacer un Pull Request

### Paso 1: Fork el Repositorio
```bash
# Haz un fork en GitHub (botón en la esquina superior derecha)
# Luego clónalo localmente
git clone https://github.com/TU_USUARIO/awesome-devops-python.git
cd awesome-devops-python
```

### Paso 2: Crea una Rama
```bash
# Crea una rama con un nombre descriptivo
git checkout -b add/nuevo-repositorio
# o
git checkout -b fix/descripcion-incorrecta
# o
git checkout -b docs/mejorar-readme
```

### Paso 3: Realiza tus Cambios
- Edita el archivo `README.md` o los que correspondan
- Mantén la consistencia con el formato existente
- Verifica que los enlaces funcionen

### Paso 4: Commit y Push
```bash
# Commit con mensaje descriptivo
git add .
git commit -m "Add: Agregar repositorio X a categoría Y"
# o
git commit -m "Fix: Corregir enlace en repositorio Z"

# Push a tu fork
git push origin add/nuevo-repositorio
```

### Paso 5: Abre un Pull Request
- Ve a tu fork en GitHub
- Haz clic en "Compare & pull request"
- Describe qué cambios hiciste y por qué
- Espera feedback

## 📋 Estándares de Formato

### Para Agregar un Repositorio
```markdown
### N. [nombre-repo](https://github.com/owner/repo) ⭐ X,XXX
Descripción breve y clara del repositorio en 1-2 líneas.
```

**Ejemplo:**
```markdown
### 31. [awesome-docker](https://github.com/veggiemonk/awesome-docker) ⭐ 35,000
Lista curada de recursos, herramientas y aplicaciones de Docker. Excelente referencia.
```

### Reglas de Formato
- ✅ Mantén el número de orden
- ✅ Incluye el enlace correcto
- ✅ Incluye el número de stars (aproximado, puede estar ~100 fuera)
- ✅ Descripción clara y concisa
- ✅ Agrupa en categoría correcta
- ✅ Mantén alfabético dentro de categorías (opcional)

## 🎯 Criterios de Aceptación

Para que tu contribución sea aceptada:

✅ **Formato correcto** - Sigue el estándar de formato  
✅ **Relevancia** - El repositorio es sobre DevOps + Python  
✅ **Activo** - El proyecto tiene actividad reciente  
✅ **Documentación clara** - Descripción y enlaces correctos  
✅ **Sin duplicados** - No existe ya en la lista  
✅ **Buen código** - El repositorio tiene buena calidad de código

## ⚠️ Normas de Conducta

- Sé respetuoso y amigable
- No hagas spam ni autopromoción excesiva
- Construye sobre ideas, no las destruyas
- Escucha feedback y estés abierto a cambios

## 💡 Consejos

1. **Comienza pequeño**: Tu primer PR puede ser una corrección de typo
2. **Lee primero**: Revisa issues abiertos para ver si alguien ya trabaja en ello
3. **Documenta**: Escribe mensajes de commit claros
4. **Prueba**: Verifica que todos los enlaces funcionen
5. **Sé paciente**: Los mantenedores necesitan tiempo para revisar

## 🐛 Reportar Bugs

Si encuentras un error en este repositorio:

1. Verifica que no esté ya reportado en Issues
2. Abre un nuevo Issue con:
   - **Título**: Descripción breve del problema
   - **Descripción**: Detalles del error, cómo reproducirlo
   - **Capturas**: Si es relevante, incluye screenshots

## 📞 Contacto

- Abre un **Issue** para preguntas
- Etiqueta con `question` o `help wanted`
- Sé específico en tu pregunta
